## Summary

F5 adds a text-selection-triggered Q&A flow to every Docusaurus page. When a student highlights ≥10 characters of prose, a small "Ask AI" popup appears near the selection. Clicking it opens a chat panel pre-filled with the selected text as grounding context. The panel streams an answer from the existing `POST /chat` endpoint, with the selected text injected as the highest-priority context passage (score=1.0) ahead of RAG-retrieved passages. No new backend endpoints are needed.

## Architecture Decisions

### AD-1: Reuse POST /chat with no backend changes
Selected text is injected as `context_passages[0]` with `score: 1.0`. The existing grounding prompt in `chat_service.py` naturally gives more weight to higher-scored passages. **Zero backend changes needed.**

### AD-2: Global mouseup/touchend listener in SelectionPopup
`SelectionPopup.tsx` uses `useEffect` to attach `mouseup` and `touchend` listeners to `document`. On each event, `window.getSelection()` is called. If selected text length ≥ 10 and selection is not inside `<code>`/`<pre>`, popup is shown. **No external library required.**

### AD-3: Extract useChatStream hook
To avoid duplicating the `handleSubmit` / SSE streaming logic from `ChatWidget.tsx`, extract it into a `useChatStream(backendUrl, initialContext?)` custom hook. Both `ChatWidget` and `SelectionChatPanel` call this hook. The hook accepts an optional `initialContext` string that gets prepended as `context_passages[0]`.

### AD-4: Panel appears near selection, not fixed position
`SelectionChatPanel` is positioned `fixed` at `{ bottom: auto, top: rect.top + scrollY + rect.height + 8px }`, clamped to viewport. On mobile (≤ 480px), it takes full-width bottom-anchored sheet layout (same pattern as ChatWidget mobile CSS).

## Implementation Phases

### Phase 1 — Extract useChatStream hook (enables reuse)
- Extract `handleSubmit` + streaming logic from `ChatWidget.tsx` into `src/components/ChatWidget/useChatStream.ts`
- `ChatWidget.tsx` calls `useChatStream(backendUrl)` — behavior unchanged
- Hook accepts optional `selectedText?: string` param; if provided, prepend as `context_passages[0]` (score=1.0) before RAG passages

### Phase 2 — SelectionPopup component
- `SelectionPopup.tsx`: `document` event listeners, `window.getSelection()` + min-length + code-block check, popup button positioned via `getBoundingClientRect()`, viewport clamping
- `SelectionPopup.module.css`: small pill-shaped popup button, positioned absolute over content

### Phase 3 — SelectionChatPanel component
- `SelectionChatPanel.tsx`: receives `selectedText: string` + `backendUrl: string` + `onClose: () => void`; renders context block + `MessageList` + input form; calls `useChatStream(backendUrl, selectedText)`
- `SelectionPopup.module.css`: add panel styles (similar to ChatWidget panel but triggered differently)

### Phase 4 — Wire into Root.tsx
- Add `<SelectionPopup backendUrl={...} />` to `Root.tsx` alongside existing `<ChatWidget />`

### Phase 5 — Tests
- `SelectionPopup.test.tsx`: selection detection, min-length suppression, code block suppression, popup visibility, dismiss on deselect
- `SelectionChatPanel.test.tsx`: renders context block, submit sends question, close clears state
