# Implementation Plan: Chat Widget

**Branch**: `004-chat-widget` | **Date**: 2026-03-15 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/004-chat-widget/spec.md`

## Summary

Build a floating React chat widget embedded in every Docusaurus page via `src/theme/Root.tsx` swizzle. The widget makes two sequential backend calls: (1) `POST /search` on the existing F3 FastAPI backend to retrieve relevant passages, then (2) `POST /chat` on a new F4 FastAPI endpoint that proxies to Gemini 2.0 Flash server-side and returns a grounded, cited answer via SSE streaming.

## Architecture Decisions

### AD-1: Two-Call Pattern (Search then Chat)

The widget makes two sequential backend calls:
1. `POST /search` → RAG retrieval (existing F3 endpoint, returns top-k passages)
2. `POST /chat` → LLM generation (new F4 endpoint, receives passages as context, streams answer)

This keeps concerns separated: retrieval is a database operation, generation is an LLM call. The frontend controls both calls, so it can show incremental loading states ("Searching textbook..." then "Generating answer...").

### AD-2: SSE Streaming Protocol

`POST /chat` returns a `text/event-stream` response. Each `data:` event is a JSON `ChatChunk` with `type` ∈ {`token`, `citations`, `error`, `done`}. Citations are emitted as a single event after the full answer text is streamed, allowing the frontend to render them without interrupting the text stream.

### AD-3: Gemini Prompt Design (Grounding)

System prompt (sent with every `/chat` request):
```
You are a helpful tutor for the "Physical AI & Humanoid Robotics" textbook.
Answer the student's question using ONLY the provided context passages.
If the context does not contain enough information, say so — do not hallucinate.
After your answer, output exactly "---CITATIONS---" on a new line, then list
each citation you used as "[$INDEX] $chapter_title — $section_heading".
```

The backend splits the streamed response on `---CITATIONS---` to separate answer from citation list, mapping citation indices back to the passed `context_passages` for metadata enrichment.

### AD-4: Docusaurus Root.tsx Injection

```tsx
// book-site/src/theme/Root.tsx
import React from 'react';
import ChatWidget from '@site/src/components/ChatWidget/ChatWidget';

export default function Root({ children }: { children: React.ReactNode }) {
  return (
    <>
      {children}
      <ChatWidget backendUrl={process.env.DOCUSAURUS_BACKEND_URL ?? 'http://localhost:8000'} />
    </>
  );
}
```

`DOCUSAURUS_BACKEND_URL` is a Docusaurus environment variable (passed at build time via `.env.development.local` or CI secrets). It is NOT a secret — it's a public URL. The Gemini key stays in the FastAPI environment.

## Non-Functional Requirements

### Security
- Gemini API key in FastAPI `.env` only — never in Docusaurus env or browser bundle.
- User query sanitized: strip leading/trailing whitespace, reject empty, cap at 1000 chars.
- LLM prompt uses context-only grounding instruction (reduces prompt injection risk).
- CORS already configured on F3 backend — `ALLOWED_ORIGINS` covers Docusaurus origin.

### Performance
- Search + Chat sequential: ~0.5s search + ~0.5s TTFT = ~1s to first visible token.
- `top_k=5` default keeps context window small (~1000 tokens) for fast generation.
- Widget CSS uses `position: fixed; z-index: 9999` — no layout reflow on open/close.

### Error Handling
| Scenario | Backend Response | Widget Display |
|----------|-----------------|----------------|
| Empty query | 422 (pre-empted client-side) | Validation hint |
| POST /search 503 | — | "Couldn't search the textbook. Try again." |
| POST /chat 503 | 503 SSE error chunk | "Something went wrong. Please try again." |
| Gemini quota exceeded | 503 with detail | Same error display |
