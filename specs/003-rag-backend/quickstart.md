# Quickstart: RAG Backend (F3)

**Purpose**: Run the RAG backend locally — ingest book content and start the search API.

**Prerequisites**:
- Python 3.11+
- Qdrant Cloud account (free tier: [qdrant.tech](https://qdrant.tech))
- Gemini API key ([aistudio.google.com](https://aistudio.google.com))
- Book content written (F2 complete)

---

## 1. Setup Environment

```bash
cd backend/
python -m venv .venv
source .venv/bin/activate       # Linux/macOS
# .venv\Scripts\activate        # Windows

pip install -r requirements.txt
```

---

## 2. Configure Secrets

```bash
cp .env.example .env
# Edit .env with your values:
```

```env
GEMINI_API_KEY=your_gemini_api_key_here
QDRANT_URL=https://your-cluster.qdrant.io
QDRANT_API_KEY=your_qdrant_api_key_here
QDRANT_COLLECTION=physical-ai-book
ALLOWED_ORIGINS=http://localhost:3000,https://hassam-rauf.github.io
PORT=8000
DOCS_DIR=../book-site/docs
```

---

## 3. Ingest Book Content

Run once to index all 19 chapters into Qdrant:

```bash
python ingest.py
```

Expected output:
```
[INFO] Found 19 markdown files in ../book-site/docs/
[INFO] Chunking files...
[INFO] Total chunks: 312
[INFO] Embedding batch 1/4 (100 chunks)...
[INFO] Embedding batch 2/4 (100 chunks)...
[INFO] Embedding batch 3/4 (100 chunks)...
[INFO] Embedding batch 4/4 (12 chunks)...
[INFO] Upserted 312 vectors to collection 'physical-ai-book'
[SUCCESS] Ingestion complete. Collection has 312 vectors.
```

Re-run after any chapter content changes. Upsert is idempotent — no duplicates created.

---

## 4. Start the Search API

```bash
uvicorn app.main:app --reload --port 8000
```

Expected output:
```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Application startup complete.
```

---

## 5. Test the API

```bash
# Health check
curl http://localhost:8000/health
# Expected: {"status":"ok","collection_name":"physical-ai-book","vector_count":312,...}

# Semantic search
curl -X POST http://localhost:8000/search \
  -H "Content-Type: application/json" \
  -d '{"query": "how does ROS 2 publish-subscribe work?", "top_k": 3}'
```

Expected search response:
```json
{
  "query": "how does ROS 2 publish-subscribe work?",
  "total": 3,
  "results": [
    {
      "text": "Topics are the most common. A publisher sends messages without knowing...",
      "source_path": "module-1/ch04-ros2-nodes-topics.md",

      "section_heading": "The Publish/Subscribe Pattern",
      "module": "module-1",
      "score": 0.912,
      "chunk_index": 2
    },
    ...
  ]
}
```

**Interactive API docs**: http://localhost:8000/docs (Swagger UI auto-generated by FastAPI)

## File Inventory

| File | Purpose |
|------|---------|
| `backend/app/main.py` | FastAPI app, CORS middleware, route registration |
| `backend/app/models.py` | Pydantic schemas: SearchRequest, SearchResponse, HealthResponse |
| `backend/app/chunker.py` | Markdown chunking: strip frontmatter + Mermaid, split on `##`, sliding window |
| `backend/app/services/gemini_service.py` | All Gemini API calls (embed_content with task types) |
| `backend/app/services/qdrant_service.py` | All Qdrant calls (create collection, upsert, search) |
| `backend/ingest.py` | CLI ingestion script: reads docs/, calls chunker + gemini + qdrant |
| `backend/requirements.txt` | Python dependencies |
| `backend/.env.example` | Environment variable template |

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `GEMINI_API_KEY not set` | Copy `.env.example` → `.env` and fill in key |
| `Qdrant connection refused` | Check `QDRANT_URL` and `QDRANT_API_KEY` in `.env` |
| `vector_count: 0` after ingest | Check ingest.py logs for embedding errors; verify `DOCS_DIR` path |
| CORS error in browser | Add your origin to `ALLOWED_ORIGINS` in `.env` |
| `top_k` validation error | Keep `top_k` between 1 and 20 |
